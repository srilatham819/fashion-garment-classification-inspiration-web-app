# Pexels Evaluation Notes

## Dataset

The evaluation set contains 80 fashion images downloaded from Pexels using the official Pexels API.

Files:

- Images: `eval/dataset/images/pexels-*.jpg`
- Labels: `eval/dataset/labels/labels.json`
- Attribution: `eval/dataset/pexels_attribution.json`
- Report: `eval/reports/eval_report.json`
- Summary: `eval/reports/eval_summary.md`

The downloaded images are grouped from five fashion-oriented searches:

- `black dress fashion`
- `red dress fashion`
- `streetwear jacket fashion`
- `white shirt fashion`
- `floral skirt fashion`

The expected labels are starter labels derived from the query buckets and are marked `needs_manual_review=true`. For a final production-grade evaluation, each image should be manually inspected and corrected.

## What Works

- The pipeline is reproducible: images, labels, attribution, predictions, accuracy, and confusion analysis are all generated by scripts.
- Occasion and location context score well for query-derived buckets where the expected value is broad.
- The local fallback now extracts image color cues and pattern-like texture cues from pixels.

## Where It Struggles

- Garment type is weak without real vision reasoning. The local fallback cannot reliably distinguish jackets, tops, skirts, and dresses from arbitrary fashion photography.
- Material is mostly speculative from pixels alone.
- Color accuracy is penalized because real photos include backgrounds, skin, shadows, and multiple garments, while expected labels are single-query labels.
- Location context cannot be inferred reliably without scene understanding or source metadata.
- Query-derived labels are not a substitute for fully manual ground truth.

## Improvements With More Time

- Run the same evaluation with OpenAI multimodal classification enabled.
- Manually review all 80 labels and correct garment type, material, color, pattern, occasion, and location context.
- Add per-field confidence and evidence snippets to the classifier output.
- Improve color extraction with person/garment segmentation so background pixels do not dominate.
- Add a second-pass evaluator or human review UI for ambiguous fields.
- Track prompt version, model version, and per-run evaluation metadata.
